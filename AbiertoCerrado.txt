En este programa, se utilizó el principio de abierto/cerrado (OCP) en la implementación del conversor de monedas para permitir la extensión del comportamiento sin modificar el código existente. Este principio se aplicó mediante la creación de una interfaz común Cmonedas, que define el método convertir, y varias clases que implementan esta interfaz para realizar diferentes conversiones de moneda (por ejemplo, DolarAEuro, EuroaDolar, etc.). Al seguir este enfoque, se pueden agregar nuevas conversiones de moneda simplemente creando nuevas clases que implementen Cmonedas, sin necesidad de alterar el código de las clases existentes, lo que mejora la extensibilidad y mantenibilidad del programa.
